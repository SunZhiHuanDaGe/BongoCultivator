# Plan 6: 系统优化与丹药重构 (System Optimization & Alchemy Redesign)

## 1. UI 与交互优化

### 1.1 状态显示迁移
- **现状**: 等级和灵石悬浮显示在小人头顶 (`info_label`)。
- **修改**:
    - 移除头顶常驻的 `info_label` 
    - 将完整状态 (境界、灵石、当前修练速度) 移至 **系统托盘 (System Tray)** 的 Tooltip 或菜单中显示。
    - 托盘图标 Hover 时显示如: `筑基期 (15%) | 灵石: 500`。

### 1.2 事件与对话显示优化
- **长文本截断优化**:
    - `info_label` 改为支持多行显示 (`setWordWrap(True)`).
    - 增加气泡背景的最大宽度和动态高度。
- **对话时长调整**:
    - 点击小人后的对话显示时间从固定的 2秒 延长至 **3秒**，或者根据文本长度动态计算显示时间 (每字 0.2s + 基础 2s)。

### 1.3 菜单文案修仙化 (Thematic Renaming)
将右键菜单和界面中的现代词汇替换为修仙术语。

| 原文 | 修改后 | 备注 |
| :--- | :--- | :--- |
| 个人属性 | **道体状态** | 包含心魔、体魄等 |
| 好感度 | **气运** | 更符合修仙设定 |
| 储物袋 | **须弥戒** | 或 “纳戒” |
| 修仙坊市 | **多宝阁** | 或 “天机阁” |
| 闭关炼丹 | **开炉炼丹** | |
| 退出 | **云游四海** | |

## 2. 掉落与丹药系统重构

### 2.1 掉落概率调整
- **现状**: 掉落率过高，导致物品泛滥。
- **调整**:
    - 大幅降低基础掉落率。
    - **历练 (Work)** 状态下的掉落判定间隔延长，或概率降低至原本的 **1/10**。
    - 掉落改为“稀缺惊喜”，而非日常刷屏。

### 2.2 动态掉落池 (Tier Design)
物品获取需与当前境界强关联。

- **逻辑**:
    - **80% 概率**: 获得当前境界匹配的材料/丹药。
    - **15% 概率**: 获得低一阶的材料 (炼丹素材)。
    - **5% 概率**: **越阶挑战**，获得高一阶的稀有材料/丹药 (惊喜感)。
- **实现**: `ItemManager` 需传入当前 `layer_index` 来决定 loot table。

### 2.3 丹药系统重构 (Alchemy 2.0)
炼丹将成为突破修为瓶颈的核心手段。

- **丹药强度定标**:
    - 设定目标：一颗高阶丹药 ≈ **3天** 挂机修为。
    - 参考数值 (基于 Plan 4 的日均 20w):
        - 一阶丹药 (炼气): +50,000 Exp (约 6小时)
        - 二阶丹药 (筑基): +600,000 Exp (约 3天)
        - 三阶丹药 (金丹): +2,000,000 Exp (约 10天, 因为金丹期长)
- **炼丹限制**:
    - **神识限制**: 只有达到对应境界才能炼制同阶丹药 (防止炼气期练出大乘丹药)。
    - **丹毒/抗药性 (可选)**: 连续服用效果递减，强迫玩家去挂机或寻找新配方。
- **配方**:
    - 炼制高阶丹药需要消耗大量低阶材料或特定稀有材料 (由“机缘”事件产出)。
